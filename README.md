# Shipd – Structural Gamification Prototype (Project Jupiter)

## Overview
This prototype repositions Shipd’s gamification from aesthetic to structural by transforming onboarding into a gated quest-based progression system tied to XP, levels, and contributor identity.

## What I Built
- Onboarding quests that unlock the broader quest system
- XP progression with animated gradient progress bar
- Level thresholds with level-up animation
- Locked/unlocked quest states
- Avatar decoration rewards tied to progression
- Persistent state (mocked via localStorage)

## Happy Path
1. Complete onboarding quests
2. Unlock main quests
3. Earn XP
4. Trigger level up
5. Customize profile decoration

## Design Rationale
Gamification should drive measurable progression, not surface-level theme. This prototype demonstrates how onboarding can become the structural entry point into contributor identity and retention.

## Tech
Frontend-only prototype. State mocked locally.
